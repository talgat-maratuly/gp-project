# GP MVP — Smoke Test

Проверка базовой цепочки **GP Service** + **GP Partner** + **API** на `http://localhost:4000`.

## Подготовка

```bash
# из корня репозитория
npm install
docker compose -f apps/api/docker-compose.yml up -d   # PostgreSQL
npm run prisma:migrate:deploy
npm run prisma:generate
npm run prisma:seed
# в логе: GP Shop: 48 товаров
```

Один терминал:

```bash
npm run dev:all
```

Или при занятых портах: `npm run kill:ports && npm run dev:all`  
См. [RUN_GP.md](./RUN_GP.md)

Проверка API:

```bash
curl http://localhost:4000/health
# ожидается {"status":"ok",...}
```

В консоли браузера Service/Partner:

- `[GP] API_URL = http://localhost:4000`
- `[GP API] GET http://localhost:4000/...`

## Типичные проблемы

| Симптом | Решение |
|---------|---------|
| `Не удалось подключиться к API` | `npm run dev:api`, проверить `VITE_API_URL=http://localhost:4000`, перезапустить Vite |
| Партнёр не видит заявки | Подуслуги при регистрации; в dev статус ACTIVE; перелогин |
| Ошибка Prisma migrate | `npm run prisma:migrate:deploy`; для `migrate dev` после переименования миграции: `prisma migrate reset` или обновить запись в `_prisma_migrations` |
| CORS | API должен разрешать :5173 и :5174 (dev по умолчанию) |
